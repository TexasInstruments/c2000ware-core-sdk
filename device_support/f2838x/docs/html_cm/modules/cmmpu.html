CM-MPU Module¶

This device has dedicated MPU for each bus-master for access protection. The Cortex-M4 comes with in-built MPU and has a separate driver. The CMMPU has been added for other AHB bus masters such as uDMA(DMPU) & Ethernet(EMPU). Access permission for each region is programmable. CM-MPU supports following protections: Read only; Full access where both Read and Write access is allowed; and No access where Read and Write access is not allowed.

CM-MPU supports 8 regions with programmable start address & size. Start address of a region must be aligned to its size. Min size of the region is limited to 1KB to optimize address decode logic, 1KB granularity should suffice requirements on uDMA and EtherNET DMA.

Regions can overlap, when regions are overlapped, a fixed priority scheme is used to resolve the access protections. Region-7 is highest priority and region-0 is lowest priority. This feature is useful to define different set of access permissions for a part of already defined bigger region. This feature can be used to work around any limitations with any aligned start address to its size.

There are 8 sub-regions for every region. Sub-regions are useful to protect memory regions smaller than 2^N KB size with fewer numbers of regions. E.g., defining protection for 6K region can be achieved with single region otherwise 3 regions would be needed.

Functions

void CMMPU_enable(uint32_t base)¶

Enables and configures the MPU for use.

This function enables either the DMA or EMAC memory protection unit. Prior to enabling the MPU, at least one region must be set by calling

CMMPU_setRegionAttributes(). Once the MPU is enabled, access violation interrupts are generated if enabled.
Parameters

  • base: is the base address of the MPU instance used.

Return

None.

void CMMPU_disable(uint32_t base)¶

Disables the MPU for use.

This function disables either the DMA or EMAC memory protection unit. When the MPU is disabled, the default memory map is used and access violation interrupts are not generated.

Parameters

  • base: is the base address of the MPU instance used.

Return

None.

void CMMPU_setRegionAttributes(uint32_t base, uint32_t regionNum, uint32_t baseAddr, uint32_t attrFlags)¶

Sets up the access rules for a specific region.

This function sets up the protection rules for a region. The region has a base address and a set of attributes including the size.

Parameters

  • base: is the base address of the MPU instance used.

  • regionNum: is the region number to set up.

  • baseAddr: is the base address of the region. It must be aligned according to the size of the region specified in attrFlags.

  • attrFlags: is a set of flags to define the attributes of the region.

The region number parameter, regionNum specifies region number to setup. Valid values for regionNum paramter are:

  • CMMPU_RGN_0

  • CMMPU_RGN_1

  • CMMPU_RGN_2

  • CMMPU_RGN_3

  • CMMPU_RGN_4

  • CMMPU_RGN_5

  • CMMPU_RGN_6

  • CMMPU_RGN_7

The base address parameter, baseAddr, must be aligned according to the size, and the size must be a power of 2.

The attrFlags parameter is the logical OR of all of the attributes of the region. It is a combination of choices for region size, read/write permissions, disabled sub-regions, and a flag to determine if the region is enabled.

The size flag determines the size of a region and must be one of the following:

  • CMMPU_RGN_SIZE_1K

  • CMMPU_RGN_SIZE_2K

  • CMMPU_RGN_SIZE_4K

  • CMMPU_RGN_SIZE_8K

  • CMMPU_RGN_SIZE_16K

  • CMMPU_RGN_SIZE_32K

  • CMMPU_RGN_SIZE_64K

  • CMMPU_RGN_SIZE_128K

  • CMMPU_RGN_SIZE_256K

  • CMMPU_RGN_SIZE_512K

  • CMMPU_RGN_SIZE_1M

  • CMMPU_RGN_SIZE_2M

  • CMMPU_RGN_SIZE_4M

  • CMMPU_RGN_SIZE_8M

  • CMMPU_RGN_SIZE_16M

  • CMMPU_RGN_SIZE_32M

  • CMMPU_RGN_SIZE_64M

  • CMMPU_RGN_SIZE_128M

  • CMMPU_RGN_SIZE_256M

  • CMMPU_RGN_SIZE_512M

  • CMMPU_RGN_SIZE_1G

  • CMMPU_RGN_SIZE_2G

  • CMMPU_RGN_SIZE_4G

The read/write access permission flag must be one of the following:

  • CMMPU_RGN_PERM_READ_ONLY- read only permission to MPU region

  • CMMPU_RGN_PERM_NO_ACCESS- disables read/write permission to MPU region

  • CMMPU_RGN_PERM_FULL_ACCESS- read/write permission to MPU region

The region is automatically divided into 8 equally-sized sub-regions by the MPU. Sub-regions can only be used in regions of size 256 bytes or larger. Any of these 8 sub-regions can be disabled, allowing for creation of “holes” in a region which can be left open, or overlaid by another region with different attributes. Any of the 8 sub-regions can be disabled with a logical OR of any of the following flags:

  • CMMPU_SUB_RGN_DISABLE_0

  • CMMPU_SUB_RGN_DISABLE_1

  • CMMPU_SUB_RGN_DISABLE_2

  • CMMPU_SUB_RGN_DISABLE_3

  • CMMPU_SUB_RGN_DISABLE_4

  • CMMPU_SUB_RGN_DISABLE_5

  • CMMPU_SUB_RGN_DISABLE_6

  • CMMPU_SUB_RGN_DISABLE_7

Finally, the region can be initially enabled or disabled with one of the following flags:

  • CMMPU_RGN_ENABLE

  • CMMPU_RGN_DISABLE

As an example, to set a region with the following attributes: size of 32 KB, read-only access, one sub-region disabled, and initially enabled, the attrFlags parameter would have the following value: (CMMPU_RGN_SIZE_32K | CMMPU_RGN_PERM_READ_ONLY | CMMPU_SUB_RGN_DISABLE_2 | CMMPU_RGN_ENABLE)

Note

This function writes to multiple registers and is not protected from interrupts. It is possible that an interrupt which accesses a region may occur while that region is in the process of being changed. The safest way to handle this is to disable a region before changing it. Refer to the discussion of this in the API Detailed Description section.

Return

None.

void CMMPU_getRegionAttributes(uint32_t base, uint32_t regionNum, uint32_t *baseAddr, uint32_t *attrFlags)¶

Gets the current settings for a specific region.

This function retrieves the configuration of a specific region. The meanings and format of the parameters is the same as that of the

CMMPU_setRegionAttributes() function.
Parameters

  • base: is the base address of the MPU instance used.

  • regionNum: is the region number to get.

  • baseAddr: points to storage for the base address of the region.

  • attrFlags: points to the attribute flags for the region.

This function can be used to save the configuration of a region for later use with the CMMPU_setRegionAttributes() function. The region’s enable state is preserved in the attributes that are saved.

Return

None.

void CMMPU_enableRegion(uint32_t base, uint32_t regionNum)¶

Enables a specific region.

This function is used to enable a memory protection region. The region should already be configured with the

CMMPU_setRegionAttributes() function. Once enabled, the memory protection rules of the region are applied and access violations cause a bus fault.
Parameters

  • base: is the base address of the MPU instance used.

  • regionNum: is the region number to enable.

Return

None.

void CMMPU_disableRegion(uint32_t base, uint32_t regionNum)¶

Disables a specific region.

This function is used to disable a previously enabled memory protection region. The region remains configured if it is not overwritten with another call to

CMMPU_setRegionAttributes(), and can be enabled again by calling CMMPU_enableRegion().
Parameters

  • base: is the base address of the MPU instance used.

  • regionNum: is the region number to disable.

Return

None.

void CMMPU_registerHandler(uint32_t intNum, void (*fnHandler)(void))¶

Registers an interrupt handler for the memory management fault.

This function sets and enables the handler to be called when the MPU generates a bus fault due to a protection region access violation.

Parameters

  • intNum: is the interrupt number to be registered.

  • fnHandler: is a pointer to the function to be called when the memory management fault occurs.

See

Interrupt_registerHandler() for important information about registering interrupt handlers.

Return

None.

void CMMPU_unregisterHandler(uint32_t intNum)¶

Unregisters an interrupt handler for the memory management fault.

This function disables and clears the handler to be called when a memory management fault occurs.

Parameters

  • intNum: is the interrupt number to be unregistered.

See

Interrupt_registerHandler() for important information about registering interrupt handlers.

Return

None.

APIs are available to set/get CM-MPU region attributes, enable/disable CM-MPU regions, enable/disable CM-MPU and register/unregister interrupt handler for CM-MPU access violation faults.

The code for this module is contained in driverlib_cm/cmmpu.c and driverlib_cm/cmmpu.h.
