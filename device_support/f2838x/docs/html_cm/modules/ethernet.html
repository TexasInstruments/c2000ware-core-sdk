Ethernet Module¶

The Ethernet API provides a set of functions for configuring and using the Ethernet MAC module. Functions are provided to setup and configure the module to transmit and receive Ethernet frames. It also provides API to configure the Management Data Input Output (MDIO) module of the Ethernet.APIs are provided for reading from and writing to the External PHY registers.

Sample callback functions for Packet Transmit completion, Packet Receive completion and a simple Packet buffer handling mechanism has been provided.The default implementations for the callback functions can be modified as per application needs. The file also provides sample Ethernet Interrupt handler routines which can be modified as per application needs.

void Ethernet_setAddend(uint32_t base, uint32_t addend)¶

Adjusts the system time update rate when using the fine correction method.

This function is used to control the rate of update of the system time when in fine update mode. Fine correction mode is selected if

ETHERNET_MAC_TIMESTAMP_CONTROL_TSCFUPDT is supplied in the config parameter passed to a previous call to Ethernet_setConfigTimestampPTP(). Fine update mode is typically used when synchronizing the local clock to the IEEE 1588 master clock. The subsecond counter is incremented by the number passed to Ethernet_setConfigTimestampPTP() in the subSecondInc parameter each time a 32-bit accumulator register generates a carry. The accumulator register is incremented by the “addend” value on each main oscillator tick, and this addend value is modified to allow fine control over the rate of change of the timestamp counter. The addend value is calculated using the ratio of the main oscillator clock rate and the desired IEEE 1588 clock rate and the subSecondInc value is set to correspond to the desired IEEE 1588 clock rate.
Parameters

  • base: is the base address of the controller.

  • addend: is the number to add to the accumulator register on each tick of the Reference PTP Clock.

As an example, using digital rollover mode and a 100 MHz Reference PTP clock with a desired IEEE 1588 clock accuracy of 50 MHz, and having made a previous call to Ethernet_setConfigTimestampPTP() with subSecondInc set to the 50 MHz clock period of 20 ns, the initial addend value would be set to 0x80000000 to generate a carry on every second Reference PTP clock tick. Because the system time updates each time the accumulator overflows, small changes in the addend value can be used to very finely control the system time rate.

Return

None.

The following describes important details and recommendations when using the ethernet API.

Ethernet_initInterface() API has to be called first and then the Ethernet_getInitConfig to get known good set of parameters for the Ethernet MAC. Ethernet_getHandle() API initializes the module with the parameters configured. It also configures a default MAC address to the interface, which can be changed using Ethernet_setMACAddr() API.

Ethernet_sendPacket() API can be used to send packet(s) from the module. When a packet is received the Receive Interrupt handler will be called. This driver also implements a simple buffer handling mechanism using the Packet Descriptor structure. The application submits a packet descriptor structure during the _sendPacket() which is returned with the Transmit completion callback for relinquishing the buffers.

During the initialization ( Ethernet_getHandle()) the Driver calls the callback function for getting the Receive buffers.

The default callback functions invoked by the Interrupt handlers are also assigned by the Ethernet_getInitConfig() Those callbacks can be modified as needed by the application. Ethernet_getHandle API also programs the default Ethernet interrupt handlers. Please use Interrupt driver library APIs to change the default interrupt handlers.

The default Receive interrupt completion handler will call the callback function to refill the receive buffers consumed. The application should provide the buffer to replenish the receive buffer pool.

Ethernet_configureMDIO(), Ethernet_readPHYRegister(), Ethernet_writePHY Register APIs can be used to configure the MDIO module and access the external PHY registers.

The code for this module is contained in driverlib_cm/ethernet.c, with driverlib_cm/ethernet.h containing the API declarations for use by applications.
