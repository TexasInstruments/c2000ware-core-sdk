I2C Module¶

The inter-integrated circuit (I2C) API provides a set of functions to configure the device’s I2C module. The driver supports operation in both master and slave mode and provides functions to initialize the module, to send and receive data, to obtain status information, and to manage interrupts.

void I2C_enableMasterIntSource(uint32_t base, uint32_t intFlags)¶

Enables individual I2C Master interrupt sources.

This function enables the indicated I2C Master interrupt sources. Only the sources that are enabled can be reflected to the processor interrupt disabled sources have no effect on the processor.

Parameters

  • base: is the base address of the I2C module.

  • intFlags: is the bit mask of the interrupt sources to be enabled.

The intFlags parameter is the logical OR of any of the following:

  • I2C_MASTER_INT_RX_FIFO_FULL - RX FIFO Full interrupt

  • I2C_MASTER_INT_TX_FIFO_EMPTY - TX FIFO Empty interrupt

  • I2C_MASTER_INT_RX_FIFO_REQ - RX FIFO Request interrupt

  • I2C_MASTER_INT_TX_FIFO_REQ - TX FIFO Request interrupt

  • I2C_MASTER_INT_ARB_LOST - Arbitration Lost interrupt

  • I2C_MASTER_INT_STOP - Stop Condition interrupt

  • I2C_MASTER_INT_START - Start Condition interrupt

  • I2C_MASTER_INT_NACK - Address/Data NACK interrupt

  • I2C_MASTER_INT_TX_DMA_DONE - TX DMA Complete interrupt

  • I2C_MASTER_INT_RX_DMA_DONE - RX DMA Complete interrupt

  • I2C_MASTER_INT_TIMEOUT - Clock Timeout interrupt

  • I2C_MASTER_INT_DATA - Data interrupt

Note

Please consult the device data sheet to determine if these features are supported.

Return

None.

void I2C_clearMasterInt(uint32_t base)¶

Clears I2C Master interrupt sources.

The I2C Master interrupt source is cleared, so that it no longer asserts. This function must be called in the interrupt handler to keep the interrupt from being triggered again immediately upon exit.

Parameters

  • base: is the base address of the I2C module.

Note

Because there is a write buffer in the Cortex-M processor, it may take several clock cycles before the interrupt source is actually cleared. Therefore, it is recommended that the interrupt source be cleared early in the interrupt handler(as opposed to the very last action) to astatic inline void returning from the interrupt handler before the interrupt source is actually cleared. Failure to do so may result in the interrupt handler being immediately reentered (because the interrupt controller still sees the interrupt source asserted).

Return

None.

void I2C_setSlaveAddress(uint32_t base, uint8_t slaveAddr, I2C_MasterRW bReceive)¶

Sets the address that the I2C Master places on the bus.

This function configures the address that the I2C Master places on the bus when initiating a transaction. When the

bReceive parameter is set to I2C_MASTER_READ, the address indicates that the I2C Master is initiating a read from the slave else set to I2C_MASTER_WRITE the address indicates that the I2C Master is initiating a write to the slave.This is done using I2C_MasterRW type.
Parameters

  • base: is the base address of the I2C module.

  • slaveAddr: 7-bit slave address

  • bReceive: flag indicating the type of communication with the slave

Return

None.

void I2C_initMaster(uint32_t base, uint32_t i2cClk, bool fast)¶

Initializes the I2C Master block.

This function initializes operation of the I2C Master block by configuring the bus speed for the master and enabling the I2C Master block.

Parameters

  • base: is the base address of the I2C module.

  • i2cClk: is the rate of the clock supplied to the I2C module. It should be provided as I2C_CLK_FREQ using the updated clk rate

  • fast: set up for fast data transfers.

The peripheral clock is the same as the processor clock.

uint32_t I2C_getMasterErr(uint32_t base)¶

Gets the error status of the I2C Master.

This function is used to obtain the error status of the Master send and receive operations.

Parameters

  • base: is the base address of the I2C module.

Return

Returns the error status, as one of I2C_MASTER_ERR_NONE, I2C_MASTER_ERR_ADDR_ACK, I2C_MASTER_ERR_DATA_ACK, I2C_MASTER_ERR_ARB_LOST or I2C_MASTER_ERR_CLK_TOUT .

Before initializing the I2C module, the user first must enable the module as a master or a slave using I2C_enableMaster() or I2C_enableSlave(). When using the API in master mode, the user must then call I2C_initMaster() which will configure the rate and duty cycle of the master clock. For slave mode, I2C_initSlave() will need to be called to set the module’s address.

For both modes, this is also the time to do any FIFO or interrupt configuration. FIFOs are configured using I2C_enableFIFO(), I2C_disableFIFO(),I2C_configureTxFIFO() and I2C_configureRxFIFO(). The functions I2C_enableMasterIntSource(), I2C_disableMasterIntSource(), I2C_enableSlaveIntSource(), I2C_disableSlaveIntSource() , I2C_getMasterIntStatus(), I2C_getSlaveIntStatus() , I2C_clearMasterIntSource, I2C_clearSlaveIntSource() are for management of interrupts. Note that the I2C module uses separate interrupt lines for its basic and FIFO interrupts although the functions to configure them are the same.

To do a transfer, for both master and slave modes, other I2C APIs should be called to configure the behavior of the module. A master will need to set I2C_setSlaveAddress() to set the address of the slave to which it will communicate. I2C_putMasterData() will place data in the transmit buffer for the master. A start condition can be sent by a master using I2C_setMasterConfig().

When receiving data, the status of data received can be checked using I2C_getSlaveIntStatus() or I2C_isSlaveIntActive(). I2C_getSlaveData() will read the data from the receive buffer and return it at the slave.

The code for this module is contained in driverlib_cm/i2c.c, with driverlib_cm/i2c.h containing the API declarations for use by applications.
