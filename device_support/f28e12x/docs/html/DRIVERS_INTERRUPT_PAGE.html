The Interrupt driver provides API to configure the PIPE module. Below are the high level features supported by the Interrupt driver.

Features Supported

  • Initialization of PIPE module and vector table
  • Default interrupt handlers
  • PIPE Global configuration
  • Interrupt channel configuration

SysConfig Features

Note
It is strongly recommend to use SysConfig where it is available instead of using direct SW API calls. This will help simplify the SW application and also catch common mistakes early in the development cycle.
  • PIPE Global configuration
  • Interrupt channel configuration
  • Generates the init code and the ISR protypes
  • Generates a file named interrupt_list.txt with the list of all interrupts in the order of priority and grouped as per the nesting groups

Features NOT Supported

  • Diagnostic features present in the PIPE module is not supported in the driver

Important Usage Guidelines

  • Make sure the ISRs are defined with the interrupt attribute. Recommended to use Sysconfig for configuring interrupt. It generates the correct attribute based on whether the interrupt is INT/RTINT/NMI.
    • attribute((interrupt("INT"))) - In case of normal INTs
    • attribute((interrupt("RTINT"))) - In case of RTINTs
    • attribute((interrupt("RTINT"))) - In case of NMI
  • Make sure the RTINT threshold is aligned with nesting group mask. This is enforced in Sysconfig UI.
