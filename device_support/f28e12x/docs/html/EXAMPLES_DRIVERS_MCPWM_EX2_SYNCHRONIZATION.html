Introduction

Synchronization

MCPWM synchronization is a crucial feature that allows multiple PWM modules to operate in a coordinated manner.

Synchronization Sources

  • External Sync
    • Input XBAR routing for external sync signals
    • Configurable sync input selection (SYNCISEL)
    • Edge-triggered synchronization events
  • Internal Sync
    • Module-to-module synchronization
    • Counter synchronization between PWM modules
    • Cascaded synchronization chains possible

Sync Mechanisms

Time Base Synchronization

  • TBCTL Register Controls
    • PHSEN (Phase Load Enable)
      • MCPWM_enablePhaseShiftLoad()
      • MCPWM_disablePhaseShiftLoad()
    • SYNCOSEL (Sync Output Select)
      • MCPWM_setSyncOutPulseMode()
    • SYNCISEL (Sync Input Select)
      • MCPWM_setSyncInPulseSource()
    • PHSDIR (Phase Direction)
      • MCPWM_setCountModeAfterSync()

Sync Out Events, see enum MCPWM_SyncOutPulseMode

Sync In Sources, see enum MCPWM_SyncInPulseSource

Phase Control Features

Phase Loading

  • TBPHS Register
    • 16-bit phase offset value
    • Loaded on sync event when PHSEN enabled

Counter Behavior

  • Post-Sync Options
    • Software configurable direction
      • Force UP counting
      • Force DOWN counting

Runtime Phase Control

  • Phase offset modifications during operation
  • Direction control after synchronization

Timing Considerations

  • Sync Latency
    • One PWM clock cycle for sync recognition
    • Counter update on next clock edge
  • Phase Accuracy
    • Resolution based on PWM clock
    • Deterministic phase relationships

Applications

  • Three-phase inverter control
  • Interleaved power converters
  • Multi-channel motor drives

Example Overview

These examples demonstrate two different approaches to implement MCPWM synchronization functionality:

  1. Driverlib Implementation (mcpwm_ex2_synchronization.c in driverlib)
  2. Direct Register/Bitfield Implementation (mcpwm_ex2_synchronization.c in examples_bitfield)

Key synchronization features demonstrated in this example include:

  1. Sync Mechanism
    • External sync input through InputXBAR
    • Module-to-module sync (MCPWM1 to MCPWM3)
    • Configurable sync events (Period match)
  2. Phase Control
    • Dynamic phase shift adjustments
    • Phase shift loading on sync events
    • Configurable counter direction after sync
  3. Timing Coordination
    • Synchronized counter updates
    • Phase-shifted operation of recieving module

Common Functionality

Both implementations configure myMCPWM0(MCPWM1) and myMCPWM1(MCPWM3) with synchronization features:

myMCPWM0 Configuration

  1. Time Base Settings:
    • Up Count Mode
    • 10KHz Frequency (15999 TBCLKs for 160MHz TBCLK)
    • Sync input enabled from InputXbar 5
    • Phase shift load enabled
    • Sync Out configured for Counter equals Period event
  2. Action Qualifier Outputs:
    • Output A1: Set High on zero, Low on CMPA match
    • Output B1: Set High on zero, Low on CMPB match
  3. Interrupt Configuration:
    • ET2 triggers every 5 "Counter = Period" events
    • Updates phase shift values at runtime

myMCPWM1 Configuration

  1. Time Base Settings:
    • Up-Down Count Mode
    • 10KHz Frequency (8000 TBCLKs for 160MHz TBCLK)
    • Sync input from myMCPWM0
    • Phase shift load enabled
    • Configurable counter direction after sync
  2. Action Qualifier Outputs:
    • Output A1: Set High on zero, Low on CMPA match
    • Output B1: Set High on zero, Low on CMPB match

External Connections

Monitor these PWM outputs on Launchpad:

  • J4 40 - GPIO0 - myMCPWM0 Output A1
  • J4 39 - GPIO1 - myMCPWM0 Output B1
  • J4 7 - GPIO12 - myMCPWM1 Output A1
  • J2 19 - GPIO29 - myMCPWM1 Output B1
