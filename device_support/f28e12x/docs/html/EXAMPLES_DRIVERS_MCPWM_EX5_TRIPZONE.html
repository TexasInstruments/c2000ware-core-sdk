Introduction

Tripzone Feature

Tripzone is a safety feature that allows PWM outputs to be quickly forced to defined states in response to fault conditions. It provides hardware-level protection for power electronic circuits.

Tripzone Features

  • Multiple Trip Sources
    • Cycle-by-cycle (CBC)
    • One-shot (OSHT)
  • Configurable Actions
    • Force outputs high/low
    • Independent control of A/B outputs
  • Interrupt Generation
    • Trip event detection
    • Status monitoring

Example Overview

This example demonstrates tripzone functionality using myMCPWM0 and myMCPWM1 modules with different trip configurations.

Module Configurations

myMCPWM0

  1. Time Base Settings:
    • Up Count Mode
    • 10 kHz Frequency (15999 period @ 160MHz TBCLK)
  2. Action Qualifier:
    • Output A1/B1: Set High on Counter Zero, Low on CMPA/CMPB up
    • Output A2/B2: Set Low on Counter Period, High on CMPA/CMPB up
    • Output A3: Set High on CMPA, Low on CMPB up
    • Output B3: Set Low on CMPA, High on CMPB up
  3. Trip Zone Configuration:
    • CBC1 as trip source
    • CBC Clear on "counter = zero" event
    • TZA configured to force Low
    • TZB configured to force High
  4. Interrupt: Enabled for CBC trip

myMCPWM1

  1. Time Base Settings:
    • Up-Down Count Mode
    • 10 kHz Frequency (8000 period @ 160MHz TBCLK)
  2. Action Qualifier:
    • Output A1: Set High on CMPA up, Low on CMPA down
    • Output B1: Set High on CMPB up, Low on CMPB down
  3. Trip Zone Configuration:
    • OSHT1 as one-shot trip source
    • TZA configured to force Low
    • TZB configured to force High
  4. Interrupt: Enabled for OST trip

Trip Signal Routing

+-----—+ +--------------—+ +----—+ +------------—+ | | | | | | -->| CBC1 (MCPWM0) | | GPIO6 |—>| INPUTXBAR_IN13 |—>|PWMXBAR|—| +------------—+ | | | | | | -->| OSHT1(MCPWM1) | +-----—+ +--------------—+ +----—+ +------------—+

External Connections

Monitor these pins on the LaunchPad:

  • J4 32 - GPIO6 - Trip Input
  • J4 40 - GPIO0 - myMCPWM0 Output A1
  • J4 39 - GPIO1 - myMCPWM0 Output B1
  • J4 38 - GPIO2 - myMCPWM0 Output A2
  • J4 37 - GPIO3 - myMCPWM0 Output B2
  • J4 36 - GPIO4 - myMCPWM0 Output A3
  • J4 35 - GPIO5 - myMCPWM0 Output B3
  • J1 7 - GPIO12 - myMCPWM1 Output A1
  • J2 19 - GPIO29 - myMCPWM1 Output B1

Important Notes

  • For external trip usage, configure GPIO6 as input
  • CBC trip clears automatically on counter zero
  • One-shot trip requires explicit clearing
  • ISR counts trips up to MAX_TRIP_ISR_COUNT before clearing one-shot condition

Steps to Run the Example

  1. Import and build the CCS project
  2. Connect an oscilloscope to observe the specified GPIO pins
  3. Launch a CCS debug session
  4. Run the executable
  5. Observe trip behavior when GPIO6 is toggled:
    • MCPWM0 outputs recover on next cycle (CBC)
    • MCPWM1 outputs remain tripped until cleared (OSHT)
