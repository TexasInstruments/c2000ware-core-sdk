The CLB driver provides API to configure the CLB module. The Configurable Logic Block (CLB) is a peripheral exclusive to C2000 devices that allows users to incorporate custom logic without the need for an external logic device such as an FPGA or CLPD. Below are the high level features supported by the CLB driver.

Features Supported

  • Support of Tiles for logic implementation.
  • Every tile has Look up tables(3 Input and 8 Output LUTs), 3 FSMs, 3 Counters, 1 High Level Controller(HLC) block and 8 Asynchronous output conditioning blocks.
  • Ouptput of one component of the Tile can be input to other.
  • Each CLB tile has 8 boundary inputs and 8 boundary outputs.
  • HLC module is used for interrupt generation ans supports upto 8 instructions.
  • The Asynchronous Output Conditioning (AOC) Block submodule enables the filtering and modification of signals going into or coming out of the CLB.

SysConfig Features

Note
It is strongly recommend to use SysConfig where it is available instead of using direct SW API calls. This will help simplify the SW application and also catch common mistakes early in the development cycle.

  • CLB tool added for adding the tiles.
  • clb.dot file can be used for diagram generation for the interconnection between the nodes.
  • clb_simulation.bat file can be used to generate a .vcd file which can be used to see the internal waveforms generated by the components of the tile.
