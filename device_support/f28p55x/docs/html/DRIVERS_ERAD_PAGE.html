The ERAD driver provides API to configure the ERAD Registers. Below are the high level features supported by the ERAD driver.

Features Supported

  • Enhanced Bus Comparator (EBC) units that can:
    • Generate hardware breakpoints
    • Generate hardware watch points
    • Generate trace tags for instruction fetch matches
    • Monitor data address and read and write buses
    • Generate event outputs that can be used by other modules.
    • Generate interrupts and non-maskable interrupts
    • Monitor program counter for currently executing instruction
  • SEC units contain counters that can enhances the debug and profiling process.
  • SEC can generate an interrupt or a watch point if the count reaches a reference value.
  • The Program Counter (PC) Trace module that can keep track of PC discontinuities or jumps, as a means to trace an entire program execution sequence over a period of time.
  • OR and AND logic to combine the events generated by different EBC and SEC units to generate new events.

SysConfig Features

Note
It is strongly recommend to use SysConfig where it is available instead of using direct SW API calls. This will help simplify the SW application and also catch common mistakes early in the development cycle.
  • Selection and configuration of ERAD Enhanced Bus Comparator (EBC), System Event Counter (SEC), AND Mask and OR Mask units.
  • Enabling and configuring ERAD interrupt.
  • Configuring ERAD Interrupt and NMI events in Error Signaling Module.

Features NOT Supported

  • EBC units does not support Chained breakpoints.
  • EBC unit cannot monitor DMA transfers.
  • Discontinuities that arise due to block repeats (RPTB instruction blocks) are not captured by the PC trace module.
