The Flash driver provides an API to configure the Flash module. Below are the high level features supported by the Flash driver.

Features Supported

  • Configure data cache, code cache, prefetch, and data preread for Flash Read Interface (FRI) 1-4
  • Configure flash wait states
  • Configure flash parity test
  • Flash initialization routine

SysConfig Features

Note
It is strongly recommend to use SysConfig where it is available instead of using direct SW API calls. This will help simplify the SW application and also catch common mistakes early in the development cycle.
  • Configure data cache, code cache, prefetch, and data preread for FRI1-FRI4
  • Configure flash wait states
  • Flash initialization routine

Features NOT Supported

NA

Important Usage Guidelines

It is important to look at your device's datasheet for information about what the required minimum flash wait state is for your selected SYSCLK frequency. See the Flash section of the TRM for more info.
