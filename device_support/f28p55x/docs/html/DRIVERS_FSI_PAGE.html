The FSI driver provides API to configure the FSI module. Below are the high level features supported by the FSI driver.

Features Supported

  • High speed communication (upto SYSCLK/2) with independent transmitter and receiver cores
  • 1/2 data lines with programmable data length and double data rate
  • Programmable frame tagging for message filtering, with support for externally triggered frame generation
  • Supports tag match notifications, hardware ping watchdog, frame error detection
  • Skew adjustment block to compensate for board and system delay mismatches
  • Hardware or software computed CRC, embedded ECC computation module
  • Support for dual interrupt lines, DMA, FSI-SPI compatibility mode

SysConfig Features

Note
It is strongly recommend to use SysConfig where it is available instead of using direct SW API calls. This will help simplify the SW application and also catch common mistakes early in the development cycle.
  • Individually configurable TX & RX modules with Loopback support
  • Configurable data lines, frame parameters
  • Specify settings for tag filtering, ping timeout, delay tapping, ECC and CRC
  • Integrated support for interrupt & RTDMA modules

Features NOT Supported

NA

Important Usage Guidelines

  • For testing high speed FSI communication, use FSI adapter board
