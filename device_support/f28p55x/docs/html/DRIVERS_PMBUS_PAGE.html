The PMBus driver provides API to configure the PMBus module. Below are the high level features supported by the PMBus driver.

Features Supported

  • Each PMBus has 2 selectable mode i.e. Controller and Target with bit rate upto 1 Mbps
  • PMBus Target mode supports features like PEC byte processing, Manual target ack mode, auto acknowledge on RX bytes
  • Zero hold time
  • Multiple PMBus use case supporting ALERT, 2-WIRE and CONTROL mode

SysConfig Features

Note
It is strongly recommend to use SysConfig where it is available instead of using direct SW API calls. This will help simplify the SW application and also catch common mistakes early in the development cycle.
  • Selecting PMBus Device mode - Controller Mode or Target Mode
  • Configuring PMBus Clock speed, module address, target address and mask
  • Specifying target configuration such as PEC processing, issue ACK on EOM, disable target address masking and auto acknowledge on RX bytes
  • Enabling and configuring PMBus interrupts and features such as Zero Hold time support

Features NOT Supported

NA

Important Usage Guidelines

  • PMBus has all pins i.e. ALERT, CTL, SDA, SCL pin qualification set as Open-drain output with pull-up enabled and no synchronization. A pull-up register is required for SDA, SCL and ALERT channel for proper functioning of PMBus module. Push-Pull and Inverted pin type is invalid for PMBus module.
