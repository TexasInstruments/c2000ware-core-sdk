Overview

  • Build Version: 25.02

  • v2 Component Library: 1.17.0
  • GC Runtime Build Version: 13.2.0

What's new for GUI Composer v25.02

  • Support for application update check when an application is installed on a PC.
  • Support for continous data save to a file.
  • Project migration to latest v3 component library.
  • Readme pop up support.
  • Updates to Device/Debug probe support to synchronize with CCS 20.1.0
  • Minor enhancements and bug fixes.

Designer

  • Designer supports projects created with v2 and v3 Component library using the same workspace. v2 based projects will not be automatically migrated to v3, thus you can continue to make updates or create projects using existing v2 Component library.
  • A v3 component library project template is available in the New project wizard.
  • Target communication components that have no visible UI elements have been added to components palette along with widget components. This approach allows to have consistent developer experience with visible and non-visible elements for configuration through properties or by editing .gui file in text edit mode. This means that for v3 projects, project wizard is not used to configure target communication.
  • Transport, model, codec and program loader can be added by dropping the components into the index.gui file from the Component Palette panel OR by copying and pasting appropriate code snippet from Connection Manager Usage help page.
  • Please note: When creating a new v3 project there is a link to target-configuration help component with pre-configured code snipptes that can be used to add pre-configured target communication.

Components

  • Component tags are changed from ti- to gc- prefix
  • The core set of common components are ported from v2 library, however, not all widgets have been migrated.
  • Components with similar functionalities are merged into one component, properties are simplified and optimized.
  • Widget styling properties accessible through widget properties have been re-worked to show only applicable properties including some widget specific properties
  • Widget styling can also be configured using native CSS styling or CSS variables exposed by the widget.

Getting started with a new project

  • Log into GUI Composer
  • Select Project | New Project from the main menu
  • In the Project wizard dialog
  • Choose Application v3 from the Project Template list
  • Fill the required information
  • You may click on link in the pop up to open Connection Manager help page that includes a number of examples of different target communication options. These would need to be pasted into .gui file (make sure to switch from graphical representation of widgets into text edit mode, but clicking on <> in main toolbar).
  • Press OK to exit and create the project
  • The GUI file editor should be opened with the default getting started example
  • Add components by dragging them from the Component Palette panel and dropping them into the index.gui file
  • Add a new widget such as LED widget or Toggle switch widget, open the Properties panel and click on the link icon beside a property to enable binding to a data source variable. Select the model id in the drop list from the previous step and enter the variable name in the textbox
  • Click on the Play toolbar button to run the application

Application Publishing

  • Works the same as for existing component library projects with same restrictions.
  • However, there is a difference in how components (e.g. widgets) are resolved. Online v2 projects (i.e. executed in a browser from Gallery) could pick a slightly newer version of components if there was a critical fix made. Offline versions (for local installation on PC) would be fixed at publish time. This created a potential difference in behavior between online and offline versions of the same app. This potential discrepancy has been eliminated in v3 based projects. Critical bug fixes may still be pushed to individual applications but online and offline apps will be in synch.

Application Installation Instructions

  • An application maybe downloaded and installed for use on a local PC without the need for internet connection. Please go to dev.ti.com/gallery and find your app. Once application tile is for the specific version is visible please click on download arrows to get a pop up for different host OSs. Choose an appropriate host link.
  • Application installer is the main download that is necessary. When executing application installer, the installer will attempt to automatically download and install correct version of GC Runtime. There may be cases where firewall or proxy settings will block automatic download. In this case please download the GCRuntime for your OS from the same tile as you app installer (bottom section of download pop up, this ensures correct version is used). Then when application installer is executed, at the GUI Composer Runtime page asking whether to Download or Install from file choose the second option and Browse to location of download GCRuntime.
